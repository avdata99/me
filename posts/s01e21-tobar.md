<p>Dennis es Ingeniero en informática y trabaja con grandes volúmenes de datos. Es gestor y desarrollador de aplicaciones del ámbito académico y usando MediaWiki para proyectos de índole colaborativa. Es integrante del Directorio de Wikimedia Chile</p>

<h3>Ideas del episodio</h3>
<ul>
  <li>Wikidata es el proyecto de datos abiertos para que las máquinas puedan identificar el conocimiento y ayudarnos a sacarle valor desde Wikipedia.</li>
  <li>Es necesario que las organizaciones que son almacenes de datos suban sus contenidos a Wikidata.</li>
  <li>Wikidata esta trabajando mucho con datos de personas, paises y ciudades. Esos datos se muestran Wikipedia a través de Wikidata</li>
  <li>Cualquier persona puede subir (hay herramientas específicas) datos a Wikidata.</li>
  <li>Wikidata tiene herramientas para vincular los elementos de los datos que se suben y los que ya existen</li>
  <li>Autores.uy subió datos de todos los autores de Wikipedia a Uruguay</li>
  <li>Wikidata prefiere conectarse a otras bases antes que recibir la subida de datos estáticos. De esta forma Wikipedia se modificaría dinámicamente a medida que cambien los recursos</li>
  <li>Los objetos deben tener identificadores únicos para identificar el proceso. Tener un URI es más importante que el formato.</li>
  <li>La biblioteca Nacional de Chile libero datos describiendo ontologías y en documentos RDF. Posiblemente haya sido el primer ente público en hacerlo en latinoamérica</li>
  <li>Wikidata genera un entorno en el que es posible hacer consultas SPARQL cruzadas hacia diferentes orígenes.</li>
  <li>Wikipedia no es el único cliente de Wikidata.</li>
  <li>Hay gente usando a modo personal Wikidata desde OpenRefine</li>
  <li>WolframAlpha usa otros recursos más formales pero tiene una lógica parecida a Wikidata</li>
  <li>La mejor forma de generar impacto con datos es pensar el aquello que mejore la calidad de vida de las personas.</li>
  <li>Hay que mirar a: Eduado Graels</li>
</ul>
